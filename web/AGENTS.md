# Miks Frontend

Vite + React 19 SPA. Stack decided in [spec 0001](../docs/specs/web/0001-frontend-architecture/index.md): TanStack Router (routing), TanStack Query (server state), Zustand (client/UI state only), React Hook Form + Zod (forms), Axios (HTTP), react-i18next (French + Malagasy), vite-plugin-pwa (offline/installable), Vitest + React Testing Library (tests), `@sentry/react` pointed at a self hosted Rustrak instance (crash reporting).

## Conventions

- **Routing**: file based routes in `src/routes/`, generated by `@tanstack/router-plugin` into `src/routeTree.gen.ts` (gitignored, regenerated by `vite build`/`vite dev`). `src/routes/__root.tsx` calls `GET /auth/me` once per app load in its loader, cached by Query; `src/routes/_authenticated.tsx` is the pathless layout guarding every protected route, reading that cached answer in `beforeLoad` rather than calling `/auth/me` again.
- **Server state always lives in TanStack Query**, never duplicated into a Zustand store. `src/stores/` is for pure client/UI state (e.g. `ui-store.ts`'s `sidebarOpen`).
- **Feature folders** (`src/features/<feature>/`): `schema.ts` (Zod), `api.ts` (Axios calls), `queries.ts` (`queryOptions` + query key factory), `hooks.ts` (`useQuery`/`useMutation` wrappers). See `src/features/auth/` for the reference shape.
- **Axios client** (`src/lib/api/client.ts`): `withCredentials: true`, no token ever read or stored in JS (the API sets httpOnly cookies). A 401 triggers a single flight `/auth/refresh` then retries the original request; refresh failure notifies `onSessionExpired` listeners (the root app navigates to `/auth/login`). Replaying an already rotated refresh token revokes the whole session (spec 0001-authentication AC-6) — this is why the refresh call is single flight, not per request.
- **Env/config**: `src/lib/config/env.ts` validates `import.meta.env` once at boot with Zod, fails loudly if a required var is missing. Add a new `VITE_*` var there, not read ad hoc from `import.meta.env` elsewhere.
- **i18n**: `src/lib/i18n/`, French (`locales/fr.json`) is the shipped language, Malagasy (`locales/mg.json`) is structured in from day one. Add a key to both files together. Amounts/dates format through `Intl`, never hand rolled.
- **Offline**: TanStack Query's cache persists to IndexedDB (`src/lib/query/persister.ts`, via `idb-keyval`) so already loaded reads survive offline; a service worker (`vite-plugin-pwa`, configured in `vite.config.ts`) makes the app installable. Mutations that change money state are never queued offline, by design.
- **UI kit**: shadcn/ui (`new-york` style) + Tailwind v4, primitives in `src/components/ui/`. Design direction: [design.md](./design.md); token values live in `src/index.css`.
- **Tests**: Vitest + React Testing Library, config in `vitest.config.ts` (separate from `vite.config.ts` to avoid the router/PWA plugins running during test collection), setup in `src/test/setup.ts`. `*.test.ts(x)` next to the file it covers.
